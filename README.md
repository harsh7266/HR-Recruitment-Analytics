# HR Analytics & Recruitment Pipeline Optimization 📊

## 📌 Project Overview
This project simulates an end-to-end data analytics workflow for a high-volume Applicant Tracking System (ATS) and HRMS platform. The objective was to analyze a dataset of 5,000+ synthetic candidate records to identify pipeline bottlenecks, evaluate sourcing channel efficiency, and provide data-driven recommendations to reduce overall Time-to-Hire.

## 🛠️ Tech Stack & Tools
* **Data Manipulation & Cleaning:** Python (Pandas, NumPy)
* **Exploratory Data Analysis (EDA):** Python (Matplotlib, Seaborn)
* **Dashboarding & Reporting:** MS Excel, Automated PDF Generation (WeasyPrint/HTML)
* **Concepts Applied:** Funnel Analysis, Conversion Rate Optimization, Statistical Distribution

## 📂 Files in this Repository
* `analysis.ipynb`: The Jupyter Notebook containing the Python code for data generation, cleaning, and EDA.
* `QuickHyre_Recruitment_Data.xlsx`: The cleaned dataset and cross-tabulated summary dashboard.
* `QuickHyre_Recruitment_Report.pdf`: The final analytical business report detailing actionable insights for stakeholders.

## 💡 Key Business Insights
1. **The Interview Bottleneck:** Nearly 40% of candidates fail at the interview stage, indicating that initial screening filters are letting underqualified candidates through to the most resource-intensive phase.
2. **Sourcing Channel ROI:** Employee Referrals yielded a >25% conversion rate, significantly outperforming generic company website applications. 
3. **Departmental Delays:** The Engineering department experiences the longest Time-to-Hire (averaging ~45 days) due to prolonged technical assessment stages.

## 🚀 Actionable Recommendations
* **Automate Technical Screening:** Integrating automated coding assessments directly into the ATS platform is projected to reduce the Engineering department's Time-to-Hire by 10-15 days.
* **Incentivize Internal Referrals:** Shift sourcing budget towards employee referral campaigns to capitalize on the highest-converting talent pipeline.
